# v0.14 Footprint Editor — Deferred-Work Cleanup Batch

**Date:** 2026-07-02
**Branch:** `feature/v0.14-footprint`
**Status:** design approved (scope + two contested decisions resolved), pending spec review

## Goal

Complete the tractable items on the v0.14 CHANGELOG "Deferred to v0.15"
list, turning four active-bar stubs in the footprint editor into real
behaviour. Explicitly **out of scope:** Break Track / Drag Track End
(needs track-segment split infrastructure and is really a PCB-routing
concept, not a footprint one) and the real wgpu 3D pipeline (TODO v2.x
per `docs/internal/docs/PCB_3D_RENDER_PLAN.md`).

All four items currently emit `FootprintActiveBarStub` (a "coming soon"
log line) from `crates/signex-app/src/library/editor/footprint/active_bar_dropdowns.rs`.

## Approved scope decisions

- **#2 filter preset** → **Full parity (b):** footprint-native
  `SelectionFilterKind` preset representation, not the schematic-typed
  `CustomFilterPreset`.
- **#3 text frame + #4 3D body** → **Minimal both:** contained schema /
  field changes, no auto-wrap, no wgpu.

## Items

### ① Move Selection by X, Y — typed-delta modal

**Current:** the "Move Selection by X, Y…" dropdown item emits
`FootprintActiveBarNudgeSelection`, which nudges the combined pad
selection by **one grid step** in +X and +Y. The geometry helper
`FootprintEditorState::nudge_pads(indices, dx, dy) -> Vec<usize>`
(state/mod.rs:406) is proven and unit-tested (three tests in
`footprint/tests.rs`), and the dispatcher body (dispatch/library.rs:5549)
already does the full `push_history` → `nudge_pads` → per-moved-pad
`mirror_move_pad_in_sketch` → `sync_pads_to_primitive` sequence.

**Design:**
- Add a small modal (title "Move Selection By", two `mm` numeric inputs
  X and Y) using the existing modal chrome + the **erasable
  string-buffer** numeric-input pattern (`reference_erasable_numeric_input`)
  so typing doesn't fight an `f64` binding.
- New message `FootprintNudgeSelectionBy { dx: f64, dy: f64 }` whose
  dispatcher reuses the exact `FootprintActiveBarNudgeSelection` body but
  takes the delta from the modal instead of `grid_step_mm`.
- Keep the plain "Move Selection" item (activates Select) and repurpose
  "Move Selection by X, Y…" to **open the modal** rather than fire the
  one-step nudge. The one-step nudge remains reachable and its dispatcher
  stays (the new handler can delegate to shared code).
- Add the modal's open flag to `view_main_for`'s `needs_overlay`
  predicate (`reference_overlay_predicate_gotcha`) or the modal renders
  from a non-canvas tab as a no-op.

**Isolation:** modal state is a single `Option<{dx_buf, dy_buf}>` field on
the footprint editor state; the geometry path is unchanged and already
tested.

### ② Custom filter preset — full footprint-native parity

**Problem found:** `active_bar::CustomFilterPreset.filters` is
`Vec<SelectionFilter>` — the **schematic** category enum
(`Components/Wires/Buses/SheetSymbols/…`). The footprint filter dropdown
toggles `SelectionFilterKind` (`Pads/Arcs/Tracks/Bodies3d/Vias/Regions/
Fills/Texts/Keepouts/Other`) against `state.selection_filter`. The two
category systems don't overlap cleanly, so the schematic presets can't be
applied to a footprint. Additionally the footprint dropdown's
"All - On / All - Off" toggle is itself still a stub
(`FootprintActiveBarStub("All filters")`).

**Design (full parity):**
- Add a **sibling** type `FootprintFilterPreset { name: String, kinds:
  Vec<SelectionFilterKind> }` (leave the schematic `CustomFilterPreset`
  untouched — keeping the two category systems cleanly separate). Persist
  in `prefs.json` under a new `footprint_filter_presets` key; add
  `read/write_footprint_filter_presets` mirroring
  `fonts::read_custom_filter_presets` and its writer.
- **Capture:** a "save current filter as preset" affordance in the
  footprint context (Properties-panel custom-filter section already exists
  for the schematic side — mirror it for footprints) snapshots
  `state.selection_filter` into a named `FootprintFilterPreset`.
- **Apply:** the preset chips in `filter_entries` fire a real
  `FootprintApplyFilterPreset(usize)` message; the dispatcher assigns the
  preset's `kinds` into `state.selection_filter` (via `SelectionFilter::
  set_all`-style bulk assignment on the footprint filter) and clears the
  menu.
- **All-On/All-Off:** wire the footprint toggle to a real bulk
  set-all/clear-all on `state.selection_filter` (drop the
  `FootprintActiveBarStub("All filters")`).
- Thread footprint presets through `PanelContext` next to
  `custom_filter_presets` so both the dropdown and the Properties panel
  read the footprint-native list in the footprint editor.

**Isolation:** footprint presets are a distinct list; schematic presets
are untouched. The footprint dropdown stops receiving the schematic
`custom_presets` and receives the footprint list instead.

### ③ True bounding-box text frame — minimal primitive change

**Current:** the "Text Frame" dropdown item arms the same `PlaceString`
place tool as "String" — it drops point silk text, no frame.

**Design (minimal):**
- Locate the footprint silk-text representation (an `FpGraphic` text
  variant in `signex-library::primitive::footprint`) and add an optional
  `frame: Option<(f32, f32)>` (width, height in mm) field
  (`#[serde(default)]` so existing `.snxfpt` files round-trip).
- New place tool `PadsTool::PlaceTextFrame` (distinct from
  `PlaceString`) whose placement **drags a rectangle** to set the frame
  size, then drops the silk string anchored to the frame.
- Render aligns/clips the string within the frame rectangle. **No
  auto-wrap / reflow** — deferred.
- Round-trip tests in `signex-library` (frame present + `None`
  back-compat) and a footprint-editor dispatch test for the place flow.

### ④ Basic 3D Body mint — CPU preview

**Current:** "3D Body" and "Extruded 3D Body" are `stub()`. The schema
already has `Footprint.body_3d: Body3D { shape: BodyShape, height_mm: f32,
… }` (single field, default `shape = ExtrudeOutline`), the bake side
(`signex-bake/src/body3d.rs`) and a CPU isometric preview
(`footprint/preview3d.rs`, renders `Footprint::body_3d`) both exist.

**Design (minimal, CPU-only):**
- "3D Body" → mint a box body: set `footprint.body_3d` to a box shape
  sized from the courtyard (or selection) bbox at a default `height_mm`
  (Altium-parity default, derived, not magic). `push_history`, mark dirty,
  clear canvas cache; the CPU preview shows it immediately.
- "Extruded 3D Body" → set `body_3d.shape = ExtrudeOutline` with a default
  `height_mm` (extrudes the fab outline / `Body3D::outline` override).
- New messages `FootprintMintBody3d` / `FootprintMintExtrudedBody3d`;
  dispatchers call into the `Body3D` builders in `signex-library`.
- **No** wgpu gizmo / interactive manipulation — stays v2.x.
- Dispatch tests assert `body_3d` is populated with the expected shape +
  height after each mint.
