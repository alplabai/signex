# Commit Derivation Checklist

Run this checklist before every renderer-related commit.

- [ ] Commit title is concise and in English.
- [ ] Every new constant includes source or design-decision rationale.
- [ ] Every new algorithm includes derivation note in code comments.
- [ ] Clean-room constraint verified: no GPL-licensed source consulted.
- [ ] No foreign parser code introduced.
- [ ] Phase note updated with verification evidence.
