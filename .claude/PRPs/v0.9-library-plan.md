# v0.9 Library Subsystem — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement Phase 0 task-by-task. Phase 1 dispatches one parallel agent per workstream; each authors its own bite-sized plan from the workstream spec before executing.

**Goal:** Land the v0.9 library subsystem (Component Editor data model, local+git library, database library + server, distributor adapters) per `docs/internal/docs/LIBRARY_PLAN.md`. Foundation in Phase 0, then 7 parallel workstreams in Phase 1. UI is **out of scope** — Caner ships UI separately.

**Architecture:**
- Two new workspace crates: `signex-library` (data model + traits + diff/search) and `signex-library-server` (axum HTTP+WS server for DB flavour).
- Foundation defines `Component`, `Revision`, `LifecycleState`, schematic/PCB embed split (`SchematicSide` / `PcbSide` / `SharedSide` / `SharedSlice`), `LibraryAdapter` trait, `DistributorAdapter` trait, `.snxpart` JSON serde, manifest TOML schema, content hashing, and PLM-reserved inert fields.
- Phase 1 fills adapter implementations (local-git, database+server, 4 distributors), diff engine, Tantivy search, AI-stub symbol from datasheet, and where-used reverse index — all reachable from Phase 0's traits.
- Object-safe trait design throughout so adapters can be `Box<dyn LibraryAdapter>` and swapped at runtime.

**Tech Stack:** Rust 2024 edition, serde + serde_json, toml, uuid v7, chrono, sha2, thiserror, anyhow. Phase 1 adds: git2, axum, tokio, tantivy, keyring, oauth2, reqwest, pdf-extract, sqlx (Postgres + SQLite).

**Scope discipline (LIBRARY_PLAN §1):** No KiCad/Altium import, no live vendor scrape (defer to v1.1), no LLM symbol synthesis (heuristic stub only), no variant-aware BOM rollup.

/// Two-segment monotonic version. Minor = compatible, major = breaking.
#[derive(Clone, Copy, Debug, PartialEq, Eq, PartialOrd, Ord, Hash, Serialize, Deserialize)]
pub struct Version {
    pub major: u32,
    pub minor: u32,
}

impl Version {
    pub fn new(major: u32, minor: u32) -> Self {
        Self { major, minor }
    }

    pub fn bump_minor(self) -> Self {
        Self::new(self.major, self.minor + 1)
    }

    pub fn bump_major(self) -> Self {
        Self::new(self.major + 1, 0)
    }
}

impl std::fmt::Display for Version {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        write!(f, "{}.{}", self.major, self.minor)
    }
}

impl std::str::FromStr for Version {
    type Err = String;
    fn from_str(s: &str) -> Result<Self, Self::Err> {
        let (maj, min) = s.split_once('.').ok_or_else(|| format!("bad version: {s}"))?;
        let major = maj.parse::<u32>().map_err(|e| e.to_string())?;
        let minor = min.parse::<u32>().map_err(|e| e.to_string())?;
        Ok(Self::new(major, minor))
    }
}

/// Subset of `SharedSide` that gets embedded in both `.snxsch` and `.snxpcb`
/// for fast BOM/fab access without forcing a cross-file dependency.
#[derive(Clone, Debug, PartialEq, Default, Serialize, Deserialize)]
pub struct SharedSlice {
    pub mpn: String,
    pub manufacturer: String,
    pub description: String,
    #[serde(default)]
    pub bom_params: ParamMap, // value, tolerance, package, etc — keyed for BOM rollup
}

impl SharedSide {
    /// Project the BOM-relevant subset for `.snxsch` / `.snxpcb` embed.
    pub fn slice_for_embed(&self) -> SharedSlice {
        let mut bom_params = ParamMap::new();
        // Conservative subset — anything else stays in shared, snapshot stays small.
        for key in ["value", "tolerance", "package", "rating"] {
            if let Some(v) = self.parameters.get(key) {
                bom_params.insert(key.to_string(), v.clone());
            }
        }
        SharedSlice {
            mpn: self.mpn.clone(),
            manufacturer: self.manufacturer.clone(),
            description: self.description.clone(),
            bom_params,
        }
    }
}

//! Deterministic content hashing for revisions.
//!
//! Hash is computed over a canonical JSON serialization of (schematic, pcb, shared)
//! using sorted-key BTreeMaps and pretty=false.

use sha2::{Digest, Sha256};
use serde::Serialize;

use crate::embed::{PcbSide, SchematicSide, SharedSide};

#[derive(Serialize)]
struct HashInput<'a> {
    schematic: &'a SchematicSide,
    pcb: &'a PcbSide,
    shared: &'a SharedSide,
}

/// Compute the canonical content hash of a revision's three sides.
pub fn hash_revision_content(
    schematic: &SchematicSide,
    pcb: &PcbSide,
    shared: &SharedSide,
) -> [u8; 32] {
    let input = HashInput { schematic, pcb, shared };
    let canon = serde_json::to_vec(&input).expect("HashInput must serialize");
    let mut hasher = Sha256::new();
    hasher.update(&canon);
    let result = hasher.finalize();
    let mut out = [0u8; 32];
    out.copy_from_slice(&result);
    out
}

/// Read a `.snxpart` file, validating schema version.
pub fn read_snxpart(path: &Path) -> Result<SnxPartFile, SnxPartError> {
    let bytes = std::fs::read(path)?;
    let file: SnxPartFile = serde_json::from_slice(&bytes)?;
    if file.schema != SCHEMA_VERSION {
        return Err(SnxPartError::UnsupportedSchema {
            found: file.schema,
            expected: SCHEMA_VERSION,
        });
    }
    Ok(file)
}

## Phase 1 — Parallel Workstreams (7 agents)

Each workstream below is a self-contained spec. Once Phase 0 lands and the foundation crate compiles, dispatch one agent per workstream; each agent authors its own bite-sized plan from the spec and executes it. Workstreams touch disjoint modules so they cannot conflict at the file level — workspace `Cargo.toml` deps are pre-declared in Task 1 to avoid lock-file races.

### WS-A: Local + git adapter

**Owner:** one agent. Spec → bite-sized plan → impl → PR.

**Files (create only):**

- `crates/signex-library/src/adapters/local_git.rs`

**Contract:**
- Implements `LibraryAdapter` for a `*.snxlib/` directory.
- Reads `manifest.toml`, walks `parts/<uuid>-<version>.snxpart`, exposes search via in-memory index over header fields.
- `save_revision` writes a new immutable file, runs `git2` add+commit using `git config user.email`.
- Drafts go to `parts/.draft/<uuid>.snxpart`.
- Locks are `.lock` files alongside the part — `parts/<uuid>.<field_set>.lock`.

**Tests:**

- Two revisions of one part — auto-bump picks minor when only `mpn` changes, major when symbol pin count changes.
- Lock file blocks a second `try_lock`; release allows it.
- Manifest workflow `review_required = true` redirects `save_revision` to a `review/<uuid>` branch.

**Deps:** `git2`, `walkdir`, `chrono` (already in workspace).

**Acceptance:** `cargo test -p signex-library --features local-git` is green; integration test exercises a full git history of 3 revisions including a major-bump.
