# Code Review — v0.8 batch (commits b8d033c · c1a2153 · 297733b)

**Reviewed**: 2026-04-25

## Summary

The batch lands a lot of correct surface work — Altium-style dirty tracking, tab right-click menu, BOM preview UI, `.snxsht` parser, project-close confirm flow. Build is green; 123 tests pass; QA harness exercises every exporter against Loratis-SN without panics.

But two of the three "killer features" don't actually work end-to-end:

1. **BOM preview's column / grouping / variant / include toggles are visual-only** — the export handler ignores them and reconstructs `BomOptions` from hardcoded defaults.
2. **Parked dirty engines are lost on reopen** — closing a tab parks the engine in `dirty_paths` (correct), but reopening via the project tree always reparses from disk, overwriting the parked engine and discarding edits. The whole point of the refactor.

Both are one-function fixes. Recommend fixing before declaring v0.8 code-complete.

### HIGH

**H1. BOM preview options are not applied to the actual export** — `crates/signex-app/src/app/handlers/menu/export.rs:335-404`

`handle_bom_preview_export` opens the file dialog, fires `Message::ExportBomFinished`, and the existing `handle_export_bom_finished` reconstructs `BomOptions` from scratch:

```rust
let opts = BomOptions {
    columns: vec![BomColumn::Name, BomColumn::Description, ...], // hardcoded
    grouping: BomGrouping::Grouped,                                // hardcoded
    include_dnp: false,                                            // hardcoded
    include_not_fitted: false,                                     // hardcoded
    active_variant: ...                                            // from project_data, not preview
    format: BomFormat::from_output_path(&save_path),               // OK
    ...
};
```

Net effect: the user toggles columns, picks a variant, switches to Flat grouping, hits Export, picks a file — and gets a Grouped 6-column Base BOM regardless. 90% of the v0.8.1 UI is dead-on-export.

**Fix**: mirror the `pending_pdf_options` pattern. In `handle_bom_preview_export`, stash `bom_preview.options.clone()` to a new `document_state.pending_bom_options: Option<BomOptions>` before kicking off the dialog. In `handle_export_bom_finished`, take the pending options if present, fall back to defaults if not. Set `format` from path extension since the user can pick a different filter.

---

**H2. Parked dirty engines are discarded on reopen** — `crates/signex-app/src/app/load_gateway.rs:147-161` + `handlers/dock/project_navigation.rs:612-616`

The dirty_paths refactor's invariant is "edits survive tab close until project save/discard". `close_tab_now` correctly keeps `engines[path]` alive when `dirty_paths.contains(&path)`. But the reopen path:

```rust
// open_project_tree_document
let schematic = kicad_parser::parse_schematic_file(&file_path)?;     // re-parse from DISK
self.open_schematic_tab(file_path, ..., schematic);                    // → apply_loaded_schematic
                                                                        // → sync_engine_from_schematic
                                                                        // → engines.insert(p, NEW engine)  ← overwrites parked
```

The parked engine in `engines[path]` is replaced by a fresh disk-parsed engine. The user's in-memory edits vanish. The red dot on the project tree row is now lying — `dirty_paths.contains(path)` is still `true`, but the engine no longer has any edits.

**Fix**: gate the disk reparse in `open_project_tree_document`:

```rust
if filename.ends_with(".kicad_sch") || filename.ends_with(".snxsch") {
    if self.document_state.engines.contains_key(&file_path)
        && self.document_state.dirty_paths.contains(&file_path)
    {
        // Restore the parked tab without re-parsing — preserves
        // in-memory edits the user made before closing the tab.
        self.attach_parked_engine_as_tab(file_path, title);
        return Ok(());
    }
    let schematic = kicad_parser::parse_schematic_file(&file_path)?;
    self.open_schematic_tab(file_path, title, schematic);
    return Ok(());
}
```

Need a new `attach_parked_engine_as_tab` helper that pushes the `TabInfo` (with `dirty: true`), sets `active_tab`, and calls `sync_active_tab` — no `engines.insert`.

### MEDIUM

**M1. SaveAll silently proceeds when a save fails** — `handlers/dock/project_navigation.rs:301-330`

```rust
match engine.save() {
    Ok(_) => { ... clear dirty ... }
    Err(err) => { log::log_error(...); }  // log + carry on
}
// Then unconditionally:
self.execute_close_project_at_tree_path(&state.tree_path)
```

If 3 of 5 sheets save and 2 fail, we log the failures and close the project anyway. The user clicked Save All expecting "save them or stop" — this silently loses work.

**Fix**: count failures, surface a `document_state.export_error = "Could not save N file(s) — project not closed"` if any failed, and skip the execute_close call when so.

**M2. Dead `_force_used` defensive function** — `crates/signex-output/src/template/format.rs:355-362`

```rust
#[allow(dead_code)]
fn _force_used() {
    let _ = parse_template;
    let _ = emit_template;
}
```

Both functions are `pub use`d via `template/mod.rs`. They're public API — won't trigger unused warnings. Safe to delete.

---

**M3. Backwards-compat shim is unreachable** — `handlers/menu/export.rs:330-333`

```rust
#[allow(dead_code)]
pub(crate) fn handle_export_bom_requested(&mut self) -> Option<Task<Message>> {
    Some(self.handle_bom_preview_open())
}
```

The dispatch layer routes `Message::ExportBomRequested` to `handle_bom_preview_open` directly. This shim is unreachable. Delete or wire it back into the dispatcher.

---

**M4. signex-app has no tests for the new state machines**

Per memory `project_signex_app_no_tests.md`, signex-app has no `#[test]` modules. The dirty-paths refactor, project-close confirm, tab context menu, and BOM preview flows ship without regression tests. Pre-existing issue; surfacing here because the new code paths are non-trivial state machines (close_tab_now × dirty_paths × engines × tabs × undocked windows) that would benefit from at least a few unit tests on the core mutators.

---

**M5. `CloseAllOthers` comment is misleading** — `handlers/document_tabs.rs:149-156`

The comment says "the kept tab's path would shift if we used indices alone, so the loop uses descending order." Actually descending order keeps *non-kept* indices stable — the kept tab's index drift is handled by `close_tab_now`'s `if active_tab >= len ... -= 1` logic, which the comment doesn't mention. Reword for accuracy.

### LOW

- **L1**. Clippy collapsible-if warnings in `dispatch/overlay.rs:216,223`, `handlers/menu/export.rs:477`, `handlers/dock/project_navigation.rs:530`, `handlers/dock/project_navigation.rs:670` (unneeded `return`). Per `feedback_ci_lint_policy.md`, clippy is relaxed during dev.
- **L2**. `Self::CONTEXT_MENU_WIDTH as f32` is a noop cast (already f32) — `view/mod.rs:3302, 3341`.
- **L3**. BOM modal's `active_label.clone() ... as_str() == variant.as_str()` could be `Some(variant) == active_variant.as_deref()`. Style only.
- **L4**. `fnum` in `template/format.rs` rounds to 6 decimals via `format!("{:.6}")` then trims. Edge case: very-fine values < 1e-6 round to "0". Acceptable for sheet-template dimensions.
- **L5**. The netlist date fix uses the title-block date string verbatim with no validation — if a user puts garbage in their title block's date field, it goes straight into the netlist. Plan §7 doesn't constrain the format. Minor.
