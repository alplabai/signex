# Signex brand assets

Canonical source SVGs for the Signex mark and wordmark. All logos derive from these.

| File | Purpose |
|---|---|
| `signex.svg` | Full mark with padded viewBox (4000×4000). Source for installer icons (.ico/.icns/PNG). |
| `signex-mark.svg` | S-only mark, tightly cropped square. Source for small UI contexts (splash, about, in-app buttons). |
| `signex-logo.svg` | Horizontal mark + lowercase "signex" wordmark. Source for splash screen, about dialog, README. |

## Gradient

All variants use the site/brand gradient:

- Stop 0 · `#3DD8FF` (electric cyan)
- Stop 1 · `#0F7A9D` (dark cyan)

## Regenerating platform bitmaps

The `signex.svg` file is the source; platform-specific bitmaps (Windows `.ico`,
macOS `.icns`, Linux PNGs) are generated by the helper script in the repo root:

```bash
installer/build-icons.sh
```

Requires one of: `rsvg-convert` (librsvg), `magick` (ImageMagick 7+), or `inkscape`.

Outputs:

- `installer/windows/signex.ico` — multi-size ICO (16, 32, 48, 64, 128, 256)
- `installer/macos/Signex.icns` — macOS icon set
- `installer/linux/signex-128.png`, `signex-256.png`, `signex-512.png` — Linux desktop-file icons

These generated files are regenerated from SVG each release and do not need
to live in git; CI runs the script before packaging.

## Runtime window icon

Wired. `crates/signex-app/src/app/bootstrap.rs::bundled_window_icon` reads
`assets/brand/generated/signex-256.png` via `include_bytes!` and passes the
result to `iced::window::open`. The embed is gated on the `has_bundled_icon`
cfg (declared in `build.rs`), so the app still builds and runs if the PNG
hasn't been generated yet — it just opens with the platform default icon.

## Windows `.exe` icon

Wired. `crates/signex-app/build.rs::embed_windows_exe_icon` uses `winres` to
compile `installer/windows/signex.ico` into `signex.exe` on Windows builds,
and also embeds `signex.manifest` for per-monitor v2 DPI awareness. Both
files have `cargo:rerun-if-changed` hints so regenerating the ICO / editing
the manifest triggers a rebuild.

Regenerate the bitmaps with `installer/build-icons.sh` (requires
rsvg-convert / magick / inkscape) or `py tools/build_icons.py` (pure-Python,
requires `pip install resvg-py pillow`).
